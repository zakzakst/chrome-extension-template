## アイデア

- 特定テキストを自動コピー
  - オプションページでセレクタのセットを設定
  - サイドパネルにセット名の一覧を表示
  - セット名をクリックすると、対応するセレクタで取得される内容が一覧表示される
- ページ目次作成
  - ページ内のheading要素を取得してサイドパネルに一覧表示
  - クリックするとその位置までスクロールする
- 画像サイズ確認（画像dimensions表示）

## プロンプト周りもろもろ

- プロンプトで値を指定したほうがいい箇所の知見をためる
  - 今回でいうとstorageの要件定義を日本語で記載したらそのままstorage keyとして利用された
- レビューすることを見据えたコードの出力プロンプト
  - 自分でコードを書いていないため、処理の流れを追うのが大変だった。生成された後に自分がレビューしやすいような出力方法を考えてプロンプトに反映できるようにしたい
  - コード自体だけでなんとかするのではなく、レビュー補助プロンプトみたいなのの利用も頭の隅におきつつ考える
- 今回レビューしづらく感じた理由の一つに「コンポーネント分割していない」があると思う
  - アプローチとしては以下の2つが考えられる
    - コンポーネント分割を指定したプロンプトを書いて出力
    - 一度コンポーネント分割せずに出力して、生成されたものに対してコンポーネント分割する
  - 個人的には、初めからコンポーネント分割を見据えたプロンプトを書くのは難しい気がする（全体像をイメージできている必要があるので）
  - AIの「素早くたたき台を作れる」「たたき台があると全体像のイメージのとっかかりになる」という良さを活かすなら「一度コンポーネント分割せずに出力して、生成されたものに対してコンポーネント分割する」のほうが良い気がする。とりあえずこっちの方向性を探ってみる
- 最初に「構成要素」をリストアップしたのは上手くいったと思う。そのあとの「挙動」を記述するのが楽だった
  - これに限らず、**まず名前を付ける**のはプロンプトを書く際にやりやすいアプローチかなと感じた
- データに関しても一意に指定できる状態だとプロンプト書きやすいか？
